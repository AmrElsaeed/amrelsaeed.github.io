Web Search Engine GitHub Repo

Description:

The Keyword Search Interface project is a web application built using the Flask framework and integrated with the Bing Web Search API. The application allows users to perform keyword-based searches, displaying relevant search results along with titles and snippets. Users can click on the titles to visit the corresponding pages. The project serves as a simplified version of mainstream search engines, demonstrating the implementation of a functional web search interface. It showcases the use of Flask for web development and integrates external data retrieval through the Bing Web Search API.

Features:

  • User-friendly interface for keyword-based web searches
  • Integration of Bing Web Search API for retrieving search results
  • Display of search results with clickable titles and descriptive snippets

Technologies Used:

  • Flask: A lightweight Python-based web framework for building web applications
  • Bing Web Search API: An external API for retrieving search results from the Bing search engine

Challenges Faced:

  1. API Integration: Integrating the Bing Web Search API with Flask required understanding API usage, authentication, and data handling.
  2. Frontend Design and User Experience: Designing an intuitive and responsive frontend for a smooth user experience was a key challenge.
  3. Data Security: Ensuring secure practices to handle the API key and protect sensitive data was crucial.
  4. Error Handling: Implementing robust error handling mechanisms to deal with potential issues during data retrieval and display.
  5. Testing and Debugging: Thoroughly testing the application to ensure functionality and performance under different scenarios was vital.

Room Reservation System GitHub Repo

I developed a room reservation system capable of booking various types of spaces, such as seminar rooms or conference halls. The project was created individually using Python for the backend and the Tkinter library for the GUI. The system requires users to input and validate five parameters: name, room number, date, start time, and end time. Input validation ensures that only clean and correctly formatted data is accepted. Errors are communicated to the user through prompt messages.

The system's core functionality involves checking the availability of the specified room during the provided time slot. If the room is available, the system books it and records the booking details in a CSV file-based database. If the room is not available, an error message notifies the user of the time conflict.

To ensure the reliability of the system, I implemented comprehensive unit test cases for each function using the Pytest framework.

The choice of Python as the primary language was driven by its ease of implementation for this project.

During the development process, I encountered several challenges. Integrating the backend with the frontend was initially challenging, as I had developed the backend first and then integrated the GUI. In hindsight, I believe it would have been more efficient to begin with the frontend.

Another notable challenge arose during the process of checking room availability. I needed to extract start and end times from the CSV file and validate if the specified time slot intersected with any existing bookings. Overcoming this obstacle required working with the datetime library and converting data to time objects, which presented a considerable learning curve.
